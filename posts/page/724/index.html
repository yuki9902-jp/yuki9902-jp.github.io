時間を持て余して

時間を持て余して、薄汚れた世の中のNEWSを読んだら心が痛くなる。

暴力だけでなく言葉も人を殺す。

字面も内容も狂気を生む。

人が人を殺す。

人が心を痛めるのは人と掛かり合うから。

優しい世界だけを見ているのは現実逃避かも知れない。

でも優しい世界を作るための努力をしなくてはいけない。

すべては難しい。

まずは自分から、そして近くの人に。

そういや自作PCを載せてなかった。

Bing - The New Search Engine by Microsoft

マイクロソフトの新しい検索エンジン「Bing」の日本語版βが公開されています。

http://www.bing.com/

トップ画面の背景が毎日変わるのと検索結果の画面が整理されて見やすいのが今のところの売りです。

本当の売りは、検索結果の最適化だそうですが、登録数が少ないこと、存在しているはずの必要な情報に行き着かないこと、などまだ最適化には程遠いのではないかと思います。

自分のサイトのURL登録をしても正しくクローラーに検出してもらえない状況です。

入り口のページに書いてある情報量やリンク数が少ないこと、robots.txt、sitemap.xmlを細かく書いていないことが原因かもしれません。

robots.txt、sitemap.xmlについては、正しく書きました。

Bingのヘルプに書かれているとおりなら、これで正しく登録されるはずなんですが。

仕事の話。

I started learning “XML DB” & “One source, Multi use”.

まさに過酷な勉強です。

XML DBがはやり始めた頃に勉強しておけばよかったかな。

基礎的な概念はわかるのですが、最適な技術、製品を見つけ出して、カスタマイズをしていくにはまったく技術も知識も足りない。

でもRDBの二次元格納には面白みを感じてはいなかったのですが、NXDB(Native XML Data Base)の階層化されない多次元構造での格納は非常に興味があります。

そんな数学的な話ではなく、現実ではXMLへのデータ変換、XMLから多種のメディアにそれぞれのメディアにあった変換出力を、いかにスムースに行うか、それが今の研究内容です。

少なくとも3年前には確実なソリューションはなかったですが、それでもさまざまな企業が模索しながらも何かしらの形を作っていました。

“One source, Multi use"は、XMLとXSLTで、Web向け、出版向けのコンテンツをStutic または Dynamic に出力していくのですが、問題はこの2分野から外れるものを如何につくるかということです。

各ターゲットのファイルフォーマットをしらべ、それに適したレンダラーの開発とXSLTの開発が必要になります。

レンダラーがすでに存在している場合は比較的簡単に開発できます。

まあ、今はそれら全体、全部を把握する作業ですかね。

携帯電話11年目

中国セルラーの時代、平成11年5月30日に携帯電話を契約してから、au by KDDI にそのまま移行し、ずっと一つの会社で使い続けています。

2年ごとに端末を換えています。

三洋→東芝→東京通信工業（ソニー）→樫尾→松下

三洋はタッチパネル部分が操作できなくなるという物理限界を迎えてお役目終了。

東芝は電源が突然切れるという問題が発生してお役目終了。

ソニーは塗装がはげはげで、さらにデザインに飽きてお役目終了。

樫尾はもっとも使いやすくタフネス携帯としては最高だったけど、さすがにデザインが古くなったのでお役目終了。

松下は現在使用中。でも表面は傷だらけ、塗装ははげはげ。それでもこれに勝るデザインの携帯が少ない。

今月あたり最新にしようかと思ってます。京セラのK002のピンクゴールドかな。

しかし料金体系がよく分からなくなったのと本体の価格が高い。。。。
